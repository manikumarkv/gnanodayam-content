# gnanodayam-content

Content for [Gnanodayam](https://gnanodayam.com), kept **separate from the app** and served to it over
the [jsDelivr](https://www.jsdelivr.com/) CDN. Editing content is a PR here — no app redeploy.

## Layout

```
details/
  index.json        ← auto-generated list of codes that have a doc (do NOT hand-edit)
  <code>.json       ← one content document per Path Explorer node (e.g. btech.json)
images/             ← images referenced by image blocks (optional)
```

A node's `code` (e.g. `btech`, `btech.cse`) comes from the app's `path_nodes` table; the app fetches
`details/<code>.json` when a student clicks "Know more".

## The content format

Each `<code>.json` is a block-based **content document**:

```jsonc
{
  "schemaVersion": 1,
  "title": "B.Tech / B.E.",
  "kicker": "Undergraduate engineering degree",
  "blocks": [ /* text · facts · callout · list · links · image · chart */ ],
  "meta": { "lastReviewed": "2026-06", "createdBy": { "kind": "ai", "name": "Claude", "at": "2026-06" } }
}
```

Full block reference, the honesty rules, and the path template are in the app repo:
`.claude/skills/content-author/SKILL.md` (also a paste-anywhere system prompt) and
`docs/dev-tech-designs/77-design.md`.

**Trust rules (enforced by the app's schema):** every estimated figure carries `source` + `date`;
AI-drafted figures are `verified: false` (shown with an "approximate" badge) until a human checks them;
links are internal (`/…`) only; image `src` is repo-relative (no external URLs). Set `meta.verifiedBy`
when you review a doc.

## The index builds itself

`details/index.json` is regenerated by `.github/workflows/build-index.yml` on every push — just add a
`details/<code>.json` and the index updates. Don't edit it by hand.
